# Погода Заполярья — PROJECT BRIEF

Региональный погодный сервис для Мурманска и Мурманской области.  
Стек: **Next.js 16 App Router · TypeScript · Bun**

---

## Идея

Готовых погодных сервисов много, но ни один не заточен под специфику Заполярья:
полярный день/ночь, арктические ветра, десятки мелких населённых пунктов — сёла,
маяки, КПП, турбазы, рыбные лагеря — которых нет на Гисметео или Яндексе.

Сервис даёт прогноз для **~70 точек Мурманской области** — от крупных городов
до отдалённых турбаз — с акцентом на местный контекст: полярный статус, метеограммы
от норвежской метеослужбы, AI-сводка «простым языком».

---

## Что сделано

### Источники данных

- **MET Norway API** (`api.met.no`) — прогноз по координатам, реальные данные без
  моков. Батчевые запросы по 4 города параллельно, чтобы не словить throttle.
  React `cache()` дедуплицирует запросы в пределах одного рендера.
- **yr.no meteogram** — официальная SVG-метеограмма на 2 суток. Проксируется через
  `/api/meteogram/[yrId]` (обходим CORS и hotlink-ограничения). При недоступности —
  деградация на собственный почасовой график (Canvas/SVG на клиенте).
- **GigaChat (Сбер)** (`ngw.devices.sberbank.ru`) — AI-сводка погоды «простым языком»
  с советом по одежде. Кэшируется через `unstable_cache` на 1 час. 3 ретрая с
  фильтрацией ответов, нарушающих правила (УФ/крем — неактуально для Заполярья).
- **Windy embed** — iframe с картой осадков, ветра и температуры. Без API ключа,
  данные ECMWF.

### Персонализация (cookie-based)

- **Node.js Proxy** (`proxy.ts`) перехватывает каждый визит на страницу
  города, обновляет два куки:
  - `pv` — количество визитов на каждый slug (`slug:count,slug:count,...`)
  - `pr` — список последних 5 посещённых slug-ов
- **Предпочитаемый город** на главной: если у какого-то города ≥5 визитов И он
  опережает второе место в 2× раза — показываем его в герое вместо Мурманска.
- **«Недавно смотрели»** — секция над сеткой городов, показывает любые недавно
  посещённые локации (не только города, но и турбазы, маяки и т.д.).
- **«смотрели»** — маленький badge в карточке города в сетке.

### Админка (`/admin`)

- **Авторизация без регистрации**: один пароль `ADMIN_PASSWORD` обменивается на
  HMAC-подписанную httpOnly-сессию (Web Crypto, stateless, без БД). `proxy.ts`
  защищает страницы `/admin/**`, API-роуты проверяют сессию сами (матчер proxy
  исключает `/api`). Секрет подписи — `ADMIN_SESSION_SECRET`.
- **Сброс кеша** — погода (MET) и ИИ-сводки (GigaChat), целиком или по городу.
  Реализован через теги Next Data Cache (`weather`/`weather:<slug>`, `ai`/`ai:<slug>`)
  и `revalidateTag(tag, "max")`.
- **Промпты ИИ** — редактирование глобального системного промпта и оверрайдов
  по городу. Шаблон с плейсхолдерами `{city}`/`{data}` (данные собирает код).
  Хранятся в `data/ai-prompts.json`; сохранение сбрасывает ИИ-кеш.
- **Точки** — добавление, редактирование и удаление кастомных локаций
  (`data/custom-cities.json`). Мерджатся со встроенным списком в серверных путях
  (`registry.ts`, server-only) и подмешиваются в поиск шапки. Кэш стора
  инвалидируется по mtime файла (render и API-роут — разные инстансы модуля).
  Защита логина: per-IP троттлинг (5 попыток / 15 мин → блок), задержка на неудаче.

### Страницы

- **Главная** — герой с погодой предпочитаемого города, AI-сводка, метеограмма,
  карта осадков Windy, сетка городов области, каталог всех точек прогноза.
- **Страница города** (`/[slug]`) — детальный прогноз: герой с температурой, AI-сводка,
  метеограмма, текущие параметры (влажность, давление, ветер, УФ и т.д.),
  карточка рассвет/закат, почасовая таблица, прогноз на 7 дней.
- **Полярный статус** — автоматически определяется полярный день / полярная ночь
  по координатам и дате, показывается badge.
- **OG-картинки** — динамические og:image (1200×630) для главной и каждой точки
  через `next/og` (файловая конвенция `opengraph-image.tsx`, мета вшивается
  автоматически). Шрифт Inter — статические TTF в `src/shared/og`, читаются с
  диска (без рантайм-фетча). PWA-манифест — сайт устанавливаемый.

### Адаптив

- Мобильная вёрстка с `full-bleed-mobile` — блоки (герой, сводка, метеограмма,
  карта) растягиваются от края до края, сохраняя скругления и бордеры.
- Шапка на мобиле: дата скрыта, поиск скрыт.
- Сетка городов: 2 колонки → 1 колонка на маленьких экранах.

## Как работает рендеринг

- **Главная (`/`)** — `force-dynamic`, читает куки на сервере, серверный компонент.
  Персонализация без клиентского мерцания (нет localStorage flash).
- **Страницы городов (`/[slug]`)** — серверный рендеринг с `unstable_cache`,
  ревалидация раз в 30 минут.
- **Метеограмма** — `"use client"`, `<img>` с `onLoad`/`onError`, деградация
  на Canvas-график пока внешний SVG не загрузился.
- **Карта Windy** — `<iframe>` напрямую, SSR-совместима.
- **Поиск** — клиентский `"use client"` с фильтрацией по CITIES.

## Что хотели добавить, но не успели

### Функциональность

- **Страница турбазы/маяка/КПП** — сейчас все ~70 локаций имеют страницы,
  но для некоторых типов (особенно маяки, КПП) можно добавить контекстный блок
  (например, «Маяк Городецкий — действующий, открыт для посещения»).
- **Анимация прогноза** — Windy умеет play/pause по часам, но iframe не даёт
  управления. Хотели сделать собственную анимацию радара через RainViewer API
  (покадровый слайдер ±12 часов).
- **Push-уведомления** — «напомни если завтра дождь». Service Worker + Web Push.
- **PWA** — манифест, оффлайн-режим с кэшем последнего прогноза.

### Технический долг

- **Тесты** — есть юнит-тесты (`bun test`): HMAC-сессия админки, валидация
  точек, рендер промпт-шаблона (41 тест). Не покрыто: `buildCityWeather`
  (парсинг MET-ответа), `visit-cookie` (логика preferred city), E2E админки.
- **Ошибки MET** — страница не падает: при недоступности MET показывается
  честный fallback (`HeroUnavailable`), метеограмма yr.no и рассвет/закат
  рендерятся независимо. Можно улучшить: показывать последние кэшированные
  данные с временем устаревания вместо «временно недоступно».
- **Аналитика** — счётчик посещений без cookies (Plausible или аналог).
- **PWA-оффлайн** — манифест есть (сайт устанавливаемый); остался service
  worker с кэшем последнего прогноза для офлайна.
